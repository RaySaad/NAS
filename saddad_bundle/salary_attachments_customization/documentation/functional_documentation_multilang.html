1. System Overview

The Salary Attachments Customization module enhances Odoo's payroll system to manage employee loan deductions with advanced features:

Key Features:

  • 📝 Loan Request Workflow: Multi-level approval process (Finance → Management → HR)
  • 💰 Payment Integration: Upfront payment processing with automatic loan confirmation
  • 📋 Installment Management: Break down loans into monthly installments with individual tracking
  • ⏸️ Postponement: Delay individual installments with distribution or new installment options
  • 🎯 Pardon System: Forgive individual installments or remaining loan amounts
  • 📊 Individual Installment Tracking: Dedicated view for managing each installment separately
  • 📊 Balance Tracking: Real-time balance calculations
  • 🔄 Payslip Integration: Automatic deduction processing
  • 📝 Audit Trail: Complete change tracking

1. نظرة عامة على النظام

وحدة تخصيص مرفقات الراتب تعزز نظام الرواتب في أودو لإدارة خصومات قروض الموظفين مع ميزات متقدمة:

الميزات الرئيسية:

  • 📝 سير عمل طلب القرض: عملية موافقة متعددة المستويات (المالية ← الإدارة ← الموارد البشرية)
  • 💰 تكامل المدفوعات: معالجة الدفع المسبق مع تأكيد القرض التلقائي
  • 📋 إدارة الأقساط: تقسيم القروض إلى أقساط شهرية مع تتبع فردي
  • ⏸️ التأجيل: تأخير الأقساط الفردية مع خيارات التوزيع أو القسط الجديد
  • 🎯 نظام العفو: إعفاء الأقساط الفردية أو مبالغ القروض المتبقية
  • 📊 تتبع الأقساط الفردية: عرض مخصص لإدارة كل قسط بشكل منفصل
  • 📊 تتبع الرصيد: حسابات الرصيد في الوقت الفعلي
  • 🔄 تكامل كشف الراتب: معالجة الخصم التلقائي
  • 📝 مسار المراجعة: تتبع كامل للتغييرات

2. Getting Started

Step 1: Access Salary Attachments

Navigate to: Employee Loan menu or Payroll → Employees → Salary Attachments

Step 2: Understanding States

  • Draft: Initial state, can be edited
  • Open: Active attachment, deductions will be processed
  • Done: Completed attachment
  • Cancelled: Cancelled attachment

2. البدء

الخطوة 1: الوصول إلى مرفقات الراتب

انتقل إلى: قروض الموظفين أو الرواتب ← الموظفون ← مرفقات الراتب

الخطوة 2: فهم الحالات

  • مسودة: الحالة الأولية، يمكن تعديلها
  • مفتوح: مرفق نشط، سيتم معالجة الخصومات
  • منجز: مرفق مكتمل
  • ملغي: مرفق ملغي

3. Loan Request Workflow

Complete Approval Workflow:

  1. Draft → Employee creates loan request
  2. Finance Approval → Finance team reviews and approves
  3. Management Approval → Management team approves
  4. HR Approval → HR team gives final approval
  5. Confirmed → Request approved, awaiting payment (if required)
  6. Completed → Payment processed, salary attachment created

Step 1: Create Loan Request

  1. Navigate to Employee Loan → Employee Loan Request
  2. Click "Create" button
  3. Fill required fields
  4. Save the record

3. سير عمل طلبات القروض

سير عمل الموافقة الكامل:

  1. مسودة ← الموظف ينشئ طلب قرض
  2. موافقة مالية ← الفريق المالي يراجع ويوافق
  3. موافقة إدارية ← فريق الإدارة يوافق
  4. موافقة الموارد البشرية ← فريق الموارد البشرية يعطي الموافقة النهائية
  5. مؤكد ← الطلب موافق عليه، في انتظار الدفع (إذا كان مطلوباً)
  6. مكتمل ← تم معالجة الدفع، تم إنشاء مرفق الراتب

الخطوة 1: إنشاء طلب قرض

  1. انتقل إلى قروض الموظفين ← طلب قرض موظف
  2. انقر على زر "إنشاء"
  3. املأ الحقول المطلوبة
  4. احفظ السجل

4. Payment Integration

Creating Payment for Loan

  1. Navigate to Accounting → Vendors → Payments
  2. Click "Create"
  3. Fill payment details
  4. Save and post the payment

4. تكامل المدفوعات

إنشاء دفع للقرض

  1. انتقل إلى المحاسبة ← الموردون ← المدفوعات
  2. انقر على "إنشاء"
  3. املأ تفاصيل الدفع
  4. احفظ وأرسل الدفع

5. Managing Salary Attachments

Create New Salary Attachment

  1. Navigate to Employee Loan menu or Payroll → Employees → Salary Attachments
  2. Click "Create" button
  3. Fill required fields
  4. Save and confirm the attachment

5. إدارة مرفقات الراتب

إنشاء مرفق راتب جديد

  1. انتقل إلى قروض الموظفين أو الرواتب ← الموظفون ← مرفقات الراتب
  2. انقر على زر "إنشاء"
  3. املأ الحقول المطلوبة
  4. احفظ وأكد المرفق

6. Managing Installments

Viewing Installments

  1. Open salary attachment record
  2. Go to "Attachment Lines" tab
  3. View all installments with dates, amounts, and status
  4. Each installment line has individual tracking and management options

Installment States

  • Unpaid (Draft): Ready for processing in next payslip
  • Paid: Successfully deducted from payslip
  • Postponed: Delayed to future with amount redistribution
  • Pardoned: Forgiven amount, excluded from future processing

6. إدارة الأقساط

عرض الأقساط

  1. افتح سجل مرفق الراتب
  2. انتقل إلى تبويب "خطوط المرفق"
  3. اعرض جميع الأقساط مع التواريخ والمبالغ والحالة
  4. كل خط قسط له خيارات تتبع وإدارة فردية

حالات الأقساط

  • غير مدفوع (مسودة): جاهز للمعالجة في كشف الراتب التالي
  • مدفوع: تم خصمه بنجاح من كشف الراتب
  • مؤجل: مؤجل للمستقبل مع إعادة توزيع المبلغ
  • معفو: مبلغ معفو، مستبعد من المعالجة المستقبلية

7. Individual Installment Operations

Accessing Individual Installments

  1. Navigate to Employee Loan → Employee Loan Installments
  2. Or access from salary attachment "Attachment Lines" tab
  3. Filter by employee, date, or state as needed

Installment Form View

Each installment line provides:

  • Attachment Details: Link to parent salary attachment
  • Employee Information: Associated employees
  • Amount & Date: Installment amount and due date
  • Payslip Link: Connected payslip when processed
  • State Tracking: Current status with history
  • Action Buttons: Postpone, pardon, and view attachment options

7. عمليات الأقساط الفردية

الوصول إلى الأقساط الفردية

  1. انتقل إلى قروض الموظفين ← أقساط قروض الموظفين
  2. أو الوصول من تبويب "خطوط المرفق" في مرفق الراتب
  3. قم بالتصفية حسب الموظف أو التاريخ أو الحالة حسب الحاجة

عرض نموذج القسط

كل خط قسط يوفر:

  • تفاصيل المرفق: رابط إلى مرفق الراتب الأصلي
  • معلومات الموظف: الموظفون المرتبطون
  • المبلغ والتاريخ: مبلغ القسط وتاريخ الاستحقاق
  • رابط كشف الراتب: كشف الراتب المرتبط عند المعالجة
  • تتبع الحالة: الحالة الحالية مع التاريخ
  • أزرار الإجراءات: خيارات التأجيل والعفو وعرض المرفق

8. Postpone & Pardon Operations

Postponing Installments

  1. From installment line, click "Request Postpone" button
  2. Postpone wizard opens with options
  3. Enter reason for postponement
  4. Choose adjustment type:
    • Distribution: Spread postponed amount across remaining unpaid installments
    • New Installment: Create new installment at the end with same amount
  5. Click "Submit" to confirm
  6. System updates installment state to "Postponed"

Pardoning Installments

  1. From installment line, click "Request Pardon" button
  2. Pardon wizard opens
  3. Enter reason for pardon
  4. Click "Submit" to confirm
  5. System updates installment state to "Pardoned"
  6. Amount is added to attachment's pardon total

8. عمليات التأجيل والعفو

تأجيل الأقساط

  1. من خط القسط، انقر على زر "طلب تأجيل"
  2. يفتح معالج التأجيل مع الخيارات
  3. أدخل سبب التأجيل
  4. اختر نوع التعديل:
    • التوزيع: توزيع المبلغ المؤجل على الأقساط المتبقية غير المدفوعة
    • قسط جديد: إنشاء قسط جديد في النهاية بنفس المبلغ
  5. انقر على "إرسال" للتأكيد
  6. النظام يحدث حالة القسط إلى "مؤجل"

عفو الأقساط

  1. من خط القسط، انقر على زر "طلب عفو"
  2. يفتح معالج العفو
  3. أدخل سبب العفو
  4. انقر على "إرسال" للتأكيد
  5. النظام يحدث حالة القسط إلى "معفو"
  6. يتم إضافة المبلغ إلى إجمالي العفو للمرفق

9. Merge Functionality

Merge Process

  1. System detects conflicting attachments
  2. Merge wizard appears automatically
  3. Choose to merge or return to existing attachment

9. وظيفة الدمج

عملية الدمج

  1. النظام يكتشف المرفقات المتضاربة
  2. معالج الدمج يظهر تلقائياً
  3. اختر الدمج أو العودة للمرفق الموجود

10. Payslip Integration

Automatic Processing

  1. When payslip is created, system checks for active attachments
  2. Finds installment lines due for that month (date matches payslip date_from)
  3. Adds deduction to payslip input lines with attachment details
  4. Links installment line to payslip input line
  5. Marks installment as "Paid" when payslip is confirmed
  6. Records actual paid amount in installment line

Special Handling

  • Postponed Installments: Automatically excluded from payslip processing
  • Pardoned Installments: Automatically excluded from payslip processing
  • Draft Installments: Amount updated in payslip if installment amount changed
  • Loan Serial Numbers: Added to input line names for tracking

10. تكامل كشف الراتب

المعالجة التلقائية

  1. عند إنشاء كشف الراتب، النظام يتحقق من المرفقات النشطة
  2. يجد خطوط الأقساط المستحقة لذلك الشهر (التاريخ يطابق تاريخ بداية كشف الراتب)
  3. يضيف الخصم إلى خطوط إدخال كشف الراتب مع تفاصيل المرفق
  4. يربط خط القسط بخط إدخال كشف الراتب
  5. يحدد القسط كـ "مدفوع" عند تأكيد كشف الراتب
  6. يسجل المبلغ المدفوع الفعلي في خط القسط

المعالجة الخاصة

  • الأقساط المؤجلة: مستبعدة تلقائياً من معالجة كشف الراتب
  • الأقساط المعفوة: مستبعدة تلقائياً من معالجة كشف الراتب
  • الأقساط المسودة: يتم تحديث المبلغ في كشف الراتب إذا تغير مبلغ القسط
  • أرقام القروض التسلسلية: تضاف إلى أسماء خطوط الإدخال للتتبع

11. Reporting & Tracking

Balance Tracking

The system automatically calculates balance amounts and tracks all changes.

Accessing Reports

  1. Loan Requests: Employee Loan → Employee Loan Request
  2. Salary Attachments: Employee Loan menu or Payroll → Employees → Salary Attachments
  3. Individual Installments: Employee Loan → Employee Loan Installments
  4. Filter by employee, state, date range, or attachment
  5. Export data for external analysis
  6. Track individual installment history through chatter

  6. تتبع تاريخ الأقساط الفردية من خلال المحادثة

12. Troubleshooting

Common Issues & Solutions

Issue: Postpone/Pardon buttons not working

Cause: Payslip already generated and confirmed for that month

Solution: Cancel the payslip, perform operation, then regenerate

Issue: Cannot access individual installment

Cause: Installments not generated yet

Solution: Confirm attachment and click "Compute Installments" first

Issue: Installment amount not updating in payslip

Cause: Installment state is postponed or pardoned

Solution: Check installment state in Employee Loan Installments menu

12. استكشاف الأخطاء وإصلاحها

المشاكل الشائعة والحلول

المشكلة: أزرار التأجيل/العفو لا تعمل

السبب: تم إنشاء وتأكيد كشف الراتب لذلك الشهر

الحل: إلغاء كشف الراتب، تنفيذ العملية، ثم إعادة الإنشاء

المشكلة: لا يمكن الوصول إلى القسط الفردي

السبب: لم يتم إنشاء الأقساط بعد

الحل: تأكيد المرفق والنقر على "حساب الأقساط" أولاً

المشكلة: مبلغ القسط لا يتحدث في كشف الراتب

السبب: حالة القسط مؤجلة أو معفوة

الحل: تحقق من حالة القسط في قائمة أقساط قروض الموظفين
