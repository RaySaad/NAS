1. Module Overview

Module Name: salary_attachments_customization

Purpose: Extends Odoo's HR Payroll module to provide enhanced salary attachment management with installment tracking, postponement, and pardon functionality.

Dependencies: hr_payroll, account

2. Models

2.1 HrSalaryAttachmentRequest (hr.salary.attachment.request)

Inherits: mail.thread

Purpose: Main model for managing employee loan requests with approval workflow.

Key Fields:

loan_serial_number (Char): Unique identifier for loan requests
employee_id (Many2one): Employee requesting the loan
total_amount (Monetary): Total loan amount
monthly_amount (Monetary): Monthly deduction amount
state (Selection): Workflow states - draft, finance_approval, management_approval, hr_approval, confirmed, completed, cancelled
loan_type_id (Many2one): Type of loan/deduction
salary_attachment_id (Many2one): Generated salary attachment

Key Methods:

send_for_approval()
Initiates approval workflow and generates serial number
action_confirm()
Creates salary attachment and confirms the request
action_done()
Completes the loan request

2.2 HrSalaryAttachmentExt (hr.salary.attachment)

Inherits: hr.salary.attachment

Purpose: Extends the base salary attachment model with installment management.

Fields:

balance_amount (Monetary): Computed remaining balance
pardon_amount (Monetary): Amount that has been pardoned
salary_attachment_request_id (Many2one): Link to original request
attachment_lines (One2many): Individual installment lines
state (Selection): Extended with 'draft' state

Methods:

generate_installments()
Creates monthly installment lines based on total and monthly amounts
action_done()
Pardons all unpaid installments and completes attachment

2.3 HrSalaryAttachmentLine (hr.salary.attachment.line)

Inherits: mail.thread

Purpose: Individual installment lines with postpone/pardon functionality.

Fields:

attachment_id (Many2one): Parent salary attachment (tracking=True)
date (Date): Installment due date (tracking=True)
amount (Float): Installment amount (tracking=True)
state (Selection): draft, paid, postponed, pardoned (tracking=True)

Methods:

action_open_postpone_request()
Opens postpone wizard with payslip validation
action_open_pardon_request()
Opens pardon wizard with payslip validation

2.4 HrSalaryAttachmentType (hr.payslip.input.type)

Inherits: hr.payslip.input.type

Purpose: Extends input types with payment configuration.

Fields:

payment_type (Selection): 'with_payment' or 'without_payment'
Determines if upfront payment is required

2.5 AccountPaymentExt (account.payment)

Inherits: account.payment

Purpose: Handles loan payments and triggers loan confirmation.

Fields:

is_loan_payment (Boolean): Identifies loan payments
loan_request_id (Many2one): Related loan request

Methods:

action_post()
Confirms loan request when payment is posted

2.6 Payroll (hr.payslip)

Inherits: hr.payslip

Purpose: Integrates salary attachments with payslip processing.

Methods:

_compute_input_line_ids()
Updates payslip inputs based on attachment line states
_update_attachment_lines()
Helper method to update attachment line states

3. Views

Key View Extensions:

  • hr_salary_attachment_view_form_ext: Adds new fields, buttons, and installment lines tab
  • view_hr_salary_attachment_line_form: Form view for installment lines with statusbar
  • view_hr_payslip_form_ext: Makes payslip input amounts readonly

4. Wizards

4.1 PardonRequest (pardon.request)

Purpose: Handles pardon requests for installments or entire attachments.

Fields:

reason (Char): Reason for pardon request
loan_line_id (Many2one): Target installment line

Methods:

action_submit() - For attachment-level pardons
action_loan_line_pardon() - For line-level pardons

4.2 PostponeRequest (postpone.request)

Purpose: Handles postponement of installments with adjustment options.

Fields:

reason (Char): Reason for postponement
adjustment_type (Selection): 'distribution' or 'new_installment'
loan_line_id (Many2one): Target installment line

Methods:

action_submit()
Processes postponement based on adjustment type

4.3 MergeAttachmentWizard (merge.attachment.wizard)

Purpose: Handles merging of conflicting salary attachments.

Fields:

old_attachment_id (Many2one): Existing attachment to merge
current_attachment_id (Many2one): New attachment

Methods:

merge_attachment()
Combines old attachment balance with new attachment

5. Security

Access rights defined in ir.model.access.csv for all models with appropriate group permissions.

⚠️ Important Notes for Developers

  • Always check payslip state before allowing postpone/pardon operations
  • Use the validation_message helper function for consistent error handling
  • Tracking is enabled on key fields - changes are logged in chatter
  • Database operations are optimized to reduce multiple writes
