1. System Overview

The Salary Attachments Customization module enhances Odoo's payroll system to manage employee loan deductions with advanced features:

Key Features:

  • 📝 Loan Request Workflow: Multi-level approval process (Finance → Management → HR)
  • 💰 Payment Integration: Upfront payment processing with automatic loan confirmation
  • 📋 Installment Management: Break down loans into monthly installments with individual tracking
  • ⏸️ Postponement: Delay individual installments with distribution or new installment options
  • 🎯 Pardon System: Forgive individual installments or remaining loan amounts
  • 📊 Individual Installment Tracking: Dedicated view for managing each installment separately
  • 📊 Balance Tracking: Real-time balance calculations
  • 🔄 Payslip Integration: Automatic deduction processing
  • 📝 Audit Trail: Complete change tracking

2. Getting Started

Step 1: Access Salary Attachments

Navigate to: Employee Loan menu or Payroll → Employees → Salary Attachments

Step 2: Understanding States

  • Draft: Initial state, can be edited
  • Open: Active attachment, deductions will be processed
  • Done: Completed attachment
  • Cancelled: Cancelled attachment

3. Loan Request Workflow

Complete Approval Workflow:

  1. Draft → Employee creates loan request
  2. Finance Approval → Finance team reviews and approves
  3. Management Approval → Management team approves
  4. HR Approval → HR team gives final approval
  5. Confirmed → Request approved, awaiting payment (if required)
  6. Completed → Payment processed, salary attachment created

Step 1: Create Loan Request

  1. Navigate to Employee Loan → Employee Loan Request
  2. Click "Create" button
  3. Fill required fields:
    • Employee: Select requesting employee
    • Request Date: Date of request
    • Loan Amount: Total requested amount
    • Monthly Deduction: Monthly installment amount
    • Loan Type: Select deduction type
    • Reason: Justification for loan
    • Start Date: When deductions should begin
  4. Save the record

Step 2: Finance Approval

  1. Finance team receives notification
  2. Reviews loan request details
  3. Clicks "Finance Approve" or "Reject"
  4. State changes to "Finance Approval"

Step 3: Management Approval

  1. Management team reviews approved request
  2. Clicks "Management Approve" or "Reject"
  3. State changes to "Management Approval"

Step 4: HR Approval

  1. HR team gives final approval
  2. Clicks "HR Approve" or "Reject"
  3. State changes to "HR Approval"

Step 5: Confirmation

  1. HR clicks "Confirm" button
  2. State changes to "Confirmed"
  3. If loan type requires payment, awaits payment processing
  4. If no payment required, salary attachment created immediately

⚠️ Important Notes:

  • Rejected requests can be reset to draft for resubmission
  • Each approval level can add comments
  • Email notifications sent at each approval stage
  • Loan types are configured with payment requirements

4. Payment Integration

Payment Processing Workflow

  1. Confirmed loan request awaits payment (if required)
  2. Finance team creates payment record
  3. Links payment to loan request via "Loan Request" field
  4. When payment is posted, system automatically:
    • Changes request state to "Completed"
    • Creates salary attachment
    • Calculates start date based on payment date

Creating Payment for Loan

  1. Navigate to Accounting → Vendors → Payments
  2. Create new payment record
  3. Select "Send Money" payment type
  4. Choose employee as partner
  5. Enter loan amount
  6. In "Loan Request" field, select the confirmed request
  7. Post the payment

Automatic Start Date Calculation

When payment is processed, the system calculates the attachment start date:

  • Payment before 15th: Deductions start same month
  • Payment after 15th: Deductions start next month
  • This ensures proper payroll processing timing

Payment Types Configuration

Loan types can be configured as:

  • With Payment: Requires upfront payment processing
  • Without Payment: Direct salary attachment creation

5. Managing Salary Attachments

Accessing from Loan Request

  1. Open completed loan request from Employee Loan → Employee Loan Request
  2. Click "Salary Attachments" button
  3. This opens the automatically created attachment

Direct Access

  1. Navigate to Employee Loan menu or Payroll → Employees → Salary Attachments
  2. Filter by employee or state as needed

Complete Workflow:

  1. Create New Attachment → Fill basic information
  2. Confirm Attachment → System validates and activates
  3. Generate Installments → Create monthly payment schedule
  4. Process Payslips → Automatic deductions begin

Step 1: Create New Salary Attachment

  1. Navigate to Employee Loan menu or Payroll → Employees → Salary Attachments
  2. Click "Create" button
  3. Fill required fields:
    • Employee: Select employee(s)
    • Description: Loan description
    • Other Input Type: Deduction type
    • Total Amount: Total loan amount
    • Monthly Amount: Monthly deduction
    • Start Date: When deductions begin
  4. Save the record

Step 2: Confirm Attachment

  1. Click "Confirm" button
  2. System checks for existing open attachments
  3. If conflicts exist, merge wizard appears
  4. State changes to "Open"

Step 3: Generate Installments

  1. Click "Compute Installments" button
  2. System creates monthly installment lines automatically
  3. Review installments in "Attachment Lines" tab
  4. Each installment line has individual tracking and management options

⚠️ Important Notes:

  • Monthly amount cannot be changed after any installment is paid
  • Attachments in draft state can be deleted
  • Confirmed attachments cannot be deleted if linked to payslips

Accessing Salary Attachments

  1. Navigate to Payroll → Employees → Salary Attachments
  2. Or click "View Salary Attachment" button from loan request

Attachment States

  • Draft: Initial state, can be edited
  • Open: Active attachment, deductions processing
  • Done: Completed attachment
  • Cancelled: Cancelled attachment

Key Actions

  • Confirm: Activates the attachment
  • Generate Installments: Creates monthly payment schedule
  • Reset to Draft: Returns to editable state (deletes installments)
  • Done: Completes and pardons remaining amounts

6. Managing Installments

Viewing Installments

  1. Open salary attachment record
  2. Go to "Attachment Lines" tab
  3. View all installments with dates, amounts, and status

Installment States

  • Unpaid (Draft): Not yet processed
  • Paid: Deducted from payslip
  • Postponed: Delayed to future
  • Pardoned: Forgiven amount

Individual Line Actions

For each installment line, you can:

  • Click "View" to open detailed form with full tracking
  • Click "Request Postpone" to delay payment
  • Click "Request Pardon" to forgive amount
  • View related payslip information when processed
  • Track state changes through chatter

7. Individual Installment Operations

Accessing Individual Installments

  1. Navigate to Employee Loan → Employee Loan Installments
  2. Or access from salary attachment "Attachment Lines" tab
  3. Filter by employee, date, or state as needed

Installment Form View

Each installment line provides:

  • Attachment Details: Link to parent salary attachment
  • Employee Information: Associated employees
  • Amount & Date: Installment amount and due date
  • Payslip Link: Connected payslip when processed
  • State Tracking: Current status with history
  • Action Buttons: Postpone, pardon, and view attachment options

State Management

Individual installments can be in the following states:

  • Unpaid (Draft): Ready for processing in next payslip
  • Paid: Successfully deducted from payslip
  • Postponed: Delayed to future with amount redistribution
  • Pardoned: Forgiven amount, excluded from future processing

8. Postpone & Pardon Operations

Postponing Installments

  1. From installment line, click "Request Postpone" button
  2. Postpone wizard opens with options
  3. Enter reason for postponement
  4. Choose adjustment type:
    • Distribution: Spread postponed amount across remaining unpaid installments
    • New Installment: Create new installment at the end with same amount
  5. Click "Submit" to confirm
  6. System updates installment state to "Postponed"
  7. Amount is redistributed according to selected option

Pardoning Installments

  1. From installment line, click "Request Pardon" button
  2. Pardon wizard opens
  3. Enter reason for pardon
  4. Click "Submit" to confirm
  5. System updates installment state to "Pardoned"
  6. Amount is added to attachment's pardon total
  7. Pardoned installments are excluded from future payslip processing

⚠️ Restrictions:

  • Cannot postpone/pardon if payslip already generated and confirmed for that month
  • Cannot postpone/pardon already paid installments
  • System validates payslip state before allowing operations
  • All operations are tracked in installment line chatter
  • Distribution option requires other unpaid installments to exist

9. Merge Functionality

When Merge Occurs

When confirming a new salary attachment, if an existing open attachment is found for the same employee and deduction type, the system will prompt to merge them.

Merge Process

  1. System detects conflicting attachments
  2. Merge wizard appears automatically
  3. Choose to merge or return to existing attachment
  4. If merging:
    • Old attachment balance added to new attachment
    • Old attachment marked as completed and archived
    • New attachment becomes active with combined amount

⚠️ Merge Notes:

  • Only unpaid installments from old attachment are considered
  • Merge notes are added to both attachments for audit trail
  • Original loan requests are linked via parent_request_id

10. Payslip Integration

Automatic Processing

  1. When payslip is created, system checks for active attachments
  2. Finds installment lines due for that month (date matches payslip date_from)
  3. Adds deduction to payslip input lines with attachment details
  4. Links installment line to payslip input line
  5. Marks installment as "Paid" when payslip is confirmed
  6. Records actual paid amount in installment line

Special Handling

  • Postponed Installments: Automatically excluded from payslip processing
  • Pardoned Installments: Automatically excluded from payslip processing
  • Draft Installments: Amount updated in payslip if installment amount changed
  • Loan Serial Numbers: Added to input line names for tracking

Payslip States Impact

  • Cancel Payslip: Installment returns to "Draft" state, payslip link removed
  • Reset to Draft: Installment maintains "Paid" state if payslip in draft
  • Delete Input Line: Installment automatically returns to "Draft" state
  • Payslip Confirmation: Installment locked as "Paid" with amount tracking

11. Reporting & Tracking

Balance Tracking

The system automatically calculates:

  • Balance Amount: Total - Paid - Pardoned
  • Paid Amount: Sum of all paid installments
  • Pardon Amount: Sum of all pardoned amounts

Change Tracking

All changes are logged in the chatter:

  • Installment state changes
  • Amount modifications
  • Date adjustments
  • Postpone/pardon operations

Accessing Reports

  1. Loan Requests: Employee Loan → Employee Loan Request
  2. Salary Attachments: Employee Loan menu or Payroll → Employees → Salary Attachments
  3. Individual Installments: Employee Loan → Employee Loan Installments
  4. Filter by employee, state, date range, or attachment
  5. Export data for external analysis
  6. Track individual installment history through chatter

12. Troubleshooting

Common Issues & Solutions

Issue: Cannot modify monthly amount

Cause: Some installments are already paid

Solution: Reset to draft (if no payslips processed) or create new attachment

Issue: Cannot delete salary attachment

Cause: Attachment is linked to payslips

Solution: Cancel related payslips first, then delete

Issue: Postpone/Pardon buttons not working

Cause: Payslip already generated and confirmed for that month

Solution: Cancel the payslip, perform operation, then regenerate

Issue: Cannot access individual installment

Cause: Installments not generated yet

Solution: Confirm attachment and click "Compute Installments" first

Issue: Installment amount not updating in payslip

Cause: Installment state is postponed or pardoned

Solution: Check installment state in Employee Loan Installments menu

Issue: Installments not appearing in payslip

Cause: Attachment not confirmed or installments postponed/pardoned

Solution: Confirm attachment and check installment states

Issue: Loan request stuck in approval

Cause: Waiting for approver action

Solution: Contact appropriate approver or reset to draft if needed

Issue: Payment not triggering attachment creation

Cause: Loan request field not set in payment

Solution: Edit payment and link to loan request, then re-post

Issue: Cannot create loan request

Cause: Missing loan type configuration

Solution: Configure salary attachment types with payment requirements

Getting Help

  • Check the chatter for change history
  • Verify attachment and installment states
  • Review payslip input lines
  • Contact system administrator for complex issues
